# [BUG] A2A - pydantic error, when AgentCard-skill-id <> endpoint url. (1.3.0)

**Issue #3897** | State: open | Created: 2025-11-12 | Updated: 2026-03-15
**Author:** chk0301
**Labels:** bug

### Description

hi, using A2A. The host agent throws below error, because it compares the endpoint with the skill.id of the agent card, defined in the remote agent's client card.

pydantic_core._pydantic_core.ValidationError: 1 validation error for AgentResponse
a2a_ids.0
  Input should be 'http://localhost:10001/.well-known/agent-card.json' [type=literal_error, input_value='Research', input_type=str]

### Steps to Reproduce

1.) create a host crew with agent using A2a

    gen_agent = Agent(
        role="Generalist",
        goal="Coordinate tasks efficiently",
        backstory="Expert at delegating to specialized agents",
        verbose=True,
        llm=llm,
        a2a=[
            A2AConfig(
                endpoint="http://localhost:10001/.well-known/agent-card.json",
                timeout=120

2.) create remote agent crew, with agent card 

        skill = AgentSkill(
            id="Research",
            name="Research",
            description="Conduct comprehensive research on any topic and provide detailed, well-structured findings with key insights and analysis",
            tags=["research", "analysis", "information-gathering", "synthesis"],
            examples=[
                "Research the latest developments in quantum computing",
                "What are the current trends in renewable energy?",
                "Analyze the impact of AI on healthcare"
            ]
        )

        agent_card = AgentCard(
            name="Research Agent",
            description="An expert research agent that can conduct thorough research on any topic and provide detailed, well-structured findings",
            url=agent_host_url,
            version='1.0.0',
            default_input_modes=ExplorerAgent.SUPPORTED_CONTENT_TYPES,
            default_output_modes=ExplorerAgent.SUPPORTED_CONTENT_TYPES,
            capabilities=capabilities,
            skills=[skill],

start the remote crew and kick off your server crew. 

### Expected behavior

to validate / assume, that agent card - skills - id is a unique ID for an agent is not realistic. It should be not used as a key. Endpoint is unique and can be used (there can be only one agent / crew listening on one endpoint), but you can have 2 agents with same skill-id's (in particular, when you consider 3rd party remote agents)
