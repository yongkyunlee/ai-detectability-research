## Build a Multi-AI Agent Workflow with Cerebras and CrewAI

In this tutorial, we'll create a simple multi-agent workflow where an agent researches emerging technologies in a given field or topic using CrewAI tools.

### Prerequisites

Ensure you have the following before getting started:

- Python 3.7 or higher
- API keys stored in your `.env` file

Install `crewai` and `crewai-tools` packages:

```
pip install crewai crewai_tools
```

### Configuring the LLM

```python
from crewai import LLM
import os

cerebras_llm = LLM(
    model="cerebras/llama3.1-70b",
    api_key=os.environ.get("CEREBRAS_API_KEY"),
    base_url="https://api.cerebras.ai/v1",
    temperature=0.5,
)
```

### Defining an Agent with Tools

In CrewAI, an agent is an autonomous entity that performs tasks based on a defined role, goal, and backstory. Agents can utilize tools and are powered by language models (LLMs).

```python
from crewai import Agent
from crewai_tools import SerperDevTool

researcher = Agent(
    role='{topic} Senior Researcher',
    goal='Uncover groundbreaking technologies in {topic} for the year 2024',
    backstory='Driven by curiosity, you explore and share the latest innovations.',
    tools=[SerperDevTool()],
    llm=cerebras_llm
)
```

- **Role:** Defines the agent's position, using `{topic}` as a placeholder for dynamic assignment.
- **Goal:** What the agent aims to achieve.
- **Backstory:** Adds depth to the agent, influencing its behavior.
- **Tools:** External tools the agent can utilize (e.g., web search using `SerperDevTool`).
- **LLM:** Specifies the language model to be used by the agent.

### Defining a Task

A Task in CrewAI represents a unit of work assigned to an agent.

```python
from crewai import Task

research_task = Task(
    description='Identify the next big trend in {topic} with pros and cons.',
    expected_output='A 3-paragraph report on emerging {topic} technologies.',
    agent=researcher
)
```

### Executing the Workflow

A Crew is a collection of agents and tasks that work together to execute a process. It serves as the orchestrator, managing the flow of tasks among agents.

```python
from crewai import Crew, Process

def main():
    crew = Crew(
        agents=[researcher],
        tasks=[research_task],
        process=Process.sequential,
        verbose=True
    )
    result = crew.kickoff(inputs={'topic': 'AI Agents'})
    print(result)

if __name__ == "__main__":
    main()
```

- **Agents:** A list of agents involved in the workflow.
- **Tasks:** A list of tasks to be executed.
- **Process:** Defines the execution strategy. `Process.sequential` means tasks are executed one after another.
- **Verbose:** When set to `True`, provides detailed output during execution.
- **kickoff():** Starts the execution of the crew's tasks, optionally accepting dynamic inputs to replace placeholders like `{topic}`.

### Full Example Code

```python
from crewai import Agent, Task, Crew, Process, LLM
from crewai_tools import SerperDevTool
import os

cerebras_llm = LLM(
    model="cerebras/llama3.1-70b",
    api_key=os.environ.get("CEREBRAS_API_KEY"),
    base_url="https://api.cerebras.ai/v1",
    temperature=0.5,
)

researcher = Agent(
    role='{topic} Senior Researcher',
    goal='Uncover groundbreaking technologies in {topic} for the year 2024',
    backstory='Driven by curiosity, you explore and share the latest innovations.',
    tools=[SerperDevTool()],
    llm=cerebras_llm
)

research_task = Task(
    description='Identify the next big trend in {topic} with pros and cons.',
    expected_output='A 3-paragraph report on emerging {topic} technologies.',
    agent=researcher
)

def main():
    crew = Crew(
        agents=[researcher],
        tasks=[research_task],
        process=Process.sequential,
        verbose=True
    )
    result = crew.kickoff(inputs={'topic': 'AI Agents'})
    print(result)

if __name__ == "__main__":
    main()
```

### Running the Agent Script

1. Ensure all prerequisites are met.
2. Save the script to a file, e.g., `crewai_demo.py`.
3. Run the script:

```
python crewai_demo.py
```

The output will be a 3-paragraph report on emerging AI agent technologies, generated by the `researcher` agent using the configured LLM and SerperDevTool for web search.
